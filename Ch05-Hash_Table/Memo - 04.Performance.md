### 以雜貨店為例，如何建立一個能立即回覆商品價格的機制
||平均情況|最差情況|
|:-:|:-:|:-:|
|搜尋|O(1)|O(n)|
|插入|O(1)|O(n)|
|刪除|O(1)|O(n)|
* 平均情況下， Hash Table 耗費的時間為 O(1)
* O(1) 稱為常數時間 ( constant time )

||Hash Table ( 平均情況 )|Hash Table ( 最差情況 )|陣列|鏈結串列|
|:-:|:-:|:-:|:-:|:-:|
|搜尋|O(1)|O(n)|O(1)|O(n)|
|插入|O(1)|O(n)|O(n)|O(1)|
|刪除|O(1)|O(n)|O(n)|O(1)|

**避免最差情況的發生，不要讓 Hash Table 產生 Collision**
1. 確保低的負載係數 ( Load Factor )
2. 選擇好的雜湊函式
